当前结论 Duration 是结果，不是瓶颈原因

先比较相同工作量下的执行时间，再用其他指标解释为什么快或慢。

先记住

  • Duration 是最终性能结果
  • Launch Stats 决定并行规模和理论上限
  • SM / Memory 先判断压力方向
  • DRAM 用来确认片外显存瓶颈
  • Occupancy 看延迟隐藏能力
  • Scheduler / Stall 找发不出指令的原因
  • Roofline 把优化方向压成复用或算力
  • 所有判断都要回到 Duration

百分比怎么读

低 资源有余量，或未被有效使用
中 存在压力，但未必是主瓶颈
高 接近上限，值得优先调查

常用采样

ncu --set speed-of-light ncu --set full ncu --metrics breakdown:sm__throughput

避免误判

不要把 SM Throughput、Memory Throughput 和 DRAM Throughput 相加，它们不是同一份预算的分项百分比。

关键指标的最短记忆法

Eligible Warps 够不够，等待卡在哪里。

L

Launch Stats

grid、block、寄存器和 Shared Memory 决定并行度上限。
