诊断

主要受计算资源限制

SM Throughput 明显最高，Memory 和 DRAM 尚有余量。继续减少显存读取可能帮助有限。

其他指标都是解释变量。若 Duration 没下降，就不能只凭某个百分比更高宣称优化成功。

Occupancy 高但仍慢

不要继续追 Occupancy，真正瓶颈可能是计算或带宽。

当前结论 Duration 是结果，不是瓶颈原因

先比较相同工作量下的执行时间，再用其他指标解释为什么快或慢。

先记住

  • Duration 才是最终性能结果
  • Throughput 看谁更接近理论峰值
  • 高利用率不自动等于高效率
  • Occupancy 用于解释延迟隐藏能力

百分比怎么读

低 资源有余量，或未被有效使用
中 存在压力，但未必是主瓶颈
高 接近上限，值得优先调查

避免误判

不要把 SM Throughput、Memory Throughput 和 DRAM Throughput 相加，它们不是同一份预算的分项百分比。
