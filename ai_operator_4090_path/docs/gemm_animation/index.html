        演进关系
        
优化没有改变数学，只改变数据如何流动

      

      

        

          01
          
Naive

          
先让 GPU 并行算起来。

          一个线程 → 一个输出
        

        

          02
          
Tiled

          
让线程合作搬运并复用数据。

          Global → Shared → 计算
        

        

          03
          
Padding

          
保持算法不变，调整 shared memory 物理布局。

          As/Bs: TILE × (TILE + 1)
        

        

          04
          
Regtile 2×2

          
让一次读取完成更多乘加。

          2 个 A × 2 个 B → 4 个输出
        

        

          05
          
Regtile 4×4

          
继续扩大线程职责，提高计算密度。

          4 个 A × 4 个 B → 16 个输出
        

        

          06
          
Float4

          
利用连续列布局，一次读取 B 的 4 个 float。

          1 个线程 → 同一行 4 列
        

        

          07
          
WMMA FP16

          
把计算粒度提升到 warp 和 Tensor Core。

          1 个 warp → 16 × 16 tile
